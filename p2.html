Fundamentals of HTML

TL;DR

Heading, Paragraph, Image, Div, Anchor, Unordered List, Ordered List HTML elements etc.

Overview

The most commonly used elements in HTML: Heading, Paragraph, Image, Div, Anchor, Unordered List, Ordered List etc.

Links

Learn more about HTML: W3Schools .
Mozilla Developer Reference: Mozilla .

Heading Element

The <h1> to <h6> HTML elements represent six levels of section headings.


<h1> is the highest section level and <h6> is the lowest.


By default, all heading elements create a block-level box in the layout, starting on a new line and taking up the full width available in their containing block.


Credits: Mozilla

Paragraph Element

The <p> HTML element represents a paragraph.


Paragraphs are usually represented in visual media as blocks of text separated from adjacent blocks by blank lines and/or first-line indentation, but HTML paragraphs can be any structural grouping of related content, such as images or form fields.


Paragraphs are block-level elements, and notably will automatically close if another block-level element is parsed before the closing </p> tag.


Credits: Mozilla

Image Element

The <img> HTML element embeds an image into the document.


The src attribute is required, and contains the path to the image you want to embed.


Credits: Mozilla

Content Division Element

The <div> HTML element is the generic container for flow content.


It has no effect on the content or layout until styled in some way using CSS (e.g. styling is directly applied to it, or some kind of layout model like Flexbox is applied to its parent element).


Credits: Mozilla

Anchor Element

The <a> HTML element (or anchor element), with its href attribute, creates a hyperlink to web pages, files, email addresses, locations in the same page, or anything else a URL can address.


Content within each <a> should indicate the link's destination. If the href attribute is present, pressing the enter key while focused on the <a> element will activate it.


Credits: Mozilla

Unordered List Element

The <ul> HTML element represents an unordered list of items, typically rendered as a bulleted list.


You can nest a <li>(List Element) to create a hollow bulleted list.


Credits: Mozilla

Ordered List Element

The <ul> HTML element represents an unordered list of items, typically rendered as a bulleted list.


You can nest a <li>(List Element) to create a hollow bulleted list.


Credits: Mozilla
